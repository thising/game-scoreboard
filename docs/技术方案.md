# 游戏积分记录板 - 技术方案

## 一、需求澄清

**核心定位**: 组织者主导的单机计分工具
- 创建者: 拥有完整读写权限
- 分享对象: 仅可查看，不可编辑
- 使用场景: 线下活动，组织者实时计分，其他人通过链接查看结果

---

## 二、问题分析

### 2.1 概要描述中的问题

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| **数据结构不完备** | 每轮数据用固定字段(user#1-#10)存储，无法扩展 | 保留固定字段（2-10人足够），简化实现 |
| **积分类型不明确** | 正分/负分混用，无明确规则 | 定义积分规则：正分=获得，负分=扣除 |
| **缺少活动状态流转** | 只有"是否结束"布尔值 | 增加状态：准备中/进行中/已结束 |
| **统计数据计算逻辑缺失** | "单轮最大积分"等未定义计算方式 | 明确定义每个统计字段的计算公式 |

### 2.2 模板参考中的问题

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| **流程不完整** | 缺少"活动结果/结算页"的完整入口 | 在活动概览页添加结束活动按钮 |
| **积分录入单位与显示不一致** | 创建时选单位(1/2/5/10/50/100)，但录入时可能混淆 | 在录入页显示当前单位，并支持切换 |
| **轮次详情缺少编辑功能** | 只能查看，无法修改历史轮次 | 增加左滑编辑/删除手势 |
| **数据持久化未考虑** | 纯前端实现，刷新丢失 | 使用 localStorage |
| **分享功能未设计** | 需要只读分享链接 | 生成分享URL（含活动ID的只读视图）|

## 二、数据模型设计

### 2.1 活动记录 (Activity) - 简化版

```typescript
interface Activity {
  id: string;                    // 唯一ID (用于分享)
  name: string;                  // 活动名称
  createdAt: number;             // 创建时间戳
  location?: string;             // 地点（可选，纯文本）
  status: 'ongoing' | 'finished'; // 状态
  scoreUnit: number;             // 积分单位: 1, 2, 5, 10, 50, 100
  players: Player[];             // 参与者列表 (2-10人)
  rounds: Round[];               // 轮次记录
  endedAt?: number;              // 结束时间
}

interface Player {
  index: number;                 // 玩家序号 0-9
  nickname: string;              // 昵称
  color: string;                 // 颜色标识 (hex)
}

interface Round {
  roundNumber: number;           // 轮次序号 (1,2,3...)
  timestamp: number;             // 记录时间
  scores: number[];              // 本轮积分数组，与players顺序对应
}
```

**简化说明**:
- 不需要 playerId，用数组索引即可（组织者创建时确定顺序）
- 不需要头像，颜色标识足够区分
- 地点简化为纯文本（定位可选，非核心功能）

### 2.2 派生数据（实时计算）- 简化版

```typescript
interface ActivityStats {
  duration: number;              // 时长（毫秒）
  totalRounds: number;           // 总轮数
  playerStats: PlayerStat[];     // 每个玩家的统计
  topPlayer: PlayerStat;         // 第一名
  maxRoundScore: {               // 单轮最大积分
    roundNumber: number;
    playerIndex: number;
    score: number;
  };
}

interface PlayerStat {
  index: number;                 // 玩家序号
  nickname: string;
  color: string;
  totalScore: number;            // 总积分
  rank: number;                  // 排名
}
```

**简化说明**:
- 去掉趋势计算（非核心）
- 去掉最佳/最差单轮（可在详情页查看）
