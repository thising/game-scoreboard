# 游戏积分记录板 - 需求规格文档

## 1. 项目概述

### 1.1 产品定位
- **类型**: 移动端Web App（组织者单机工具）
- **用户**: 活动组织者（唯一操作者）
- **场景**: 线下游戏/活动实时计分
- **分享**: 参与者通过长图查看结果

### 1.2 核心原则
- 组织者主导：只有创建者可编辑
- 实时计分：每轮录入，即时更新排名
- 离线可用：纯前端，数据存于本机
- 自动结束：超时自动标记，防止遗忘

## 2. 功能需求

### 2.1 首页
- 显示"进行中"活动（按时间倒序）
- 显示"已结束"活动（最近10条）
- 创建新活动按钮
- 点击活动进入概览页

### 2.2 创建活动
- **活动名称**: 必填，文本输入
- **地点**: 可选，文本输入
- **参与人数**: 滑块选择，2-10人，默认4人
- **玩家昵称**: 动态生成输入框，必填
- **玩家颜色**: 自动分配（10种预设颜色循环）
- **积分单位**: 单选，选项：1, 2, 5, 10, 50, 100，默认1
- **开始记录**: 按钮，创建后跳转活动概览页

### 2.3 活动概览页
- **活动信息**: 名称、时间、地点、状态标签
- **统计卡片**:
  - 时长（实时更新）
  - 总轮数
  - 当前第一名
  - 单轮最高分（玩家+分数）
- **排行榜**: 玩家列表，显示颜色标记、昵称、总积分、排名
- **最近轮次**: 显示最近3轮，每轮显示轮次号、时间、各玩家得分
- **操作按钮**:
  - 录入本轮（进入积分录入页）
  - 查看全部轮次（进入轮次详情页）
  - 结束活动（二次确认）
  - 分享长图（生成图片）

### 2.4 积分录入页
- **当前单位显示**: 显示当前积分单位
- **单位切换**: 支持临时切换（不影响活动设置）
- **玩家列表**: 每个玩家显示颜色、昵称、当前总分
- **计分器**: +/- 按钮，按当前单位增减
- **平衡校验**: 保存前校验所有玩家积分之和是否为0
  - 不平衡：禁止保存，提示"本轮积分必须平衡"
- **保存**: 保存本轮，返回概览页

### 2.5 轮次详情页
- **轮次列表**: 显示所有轮次，每轮显示序号、时间、各玩家得分
- **搜索/跳转**: 支持跳转到指定轮次
- **编辑**: 左滑显示编辑按钮，可修改该轮分数（重新校验平衡）
- **删除**: 左滑显示删除按钮，二次确认后删除

### 2.6 活动结果页（结束后的概览）
- 与概览页相同，但状态为"已结束"
- 显示最终排名
- 隐藏"录入"、"结束"按钮
- 保留"分享长图"按钮

### 2.7 分享长图
- **内容**:
  - 活动名称、时间、地点
  - 统计摘要（时长、总轮数）
  - 排行榜（带颜色标记）
- **不包含**: 轮次明细
- **生成**: 使用 html2canvas
- **保存**: 自动下载图片到本地

## 3. 数据需求

### 3.1 数据模型

```typescript
interface Activity {
  id: string;                    // 唯一ID (UUID)
  name: string;                  // 活动名称
  createdAt: number;             // 创建时间戳
  location?: string;             // 地点（可选）
  status: 'ongoing' | 'finished'; // 状态
  scoreUnit: number;             // 积分单位: 1, 2, 5, 10, 50, 100
  players: Player[];             // 参与者列表 (2-10人)
  rounds: Round[];               // 轮次记录
  endedAt?: number;              // 结束时间
  lastRoundAt?: number;          // 最后一轮时间（用于自动结束判断）
}

interface Player {
  index: number;                 // 玩家序号 0-9
  nickname: string;              // 昵称
  color: string;                 // 颜色标识 (hex)
}

interface Round {
  roundNumber: number;           // 轮次序号 (1,2,3...)
  timestamp: number;             // 记录时间
  scores: number[];              // 本轮积分数组，与players顺序对应
}
```

### 3.2 派生数据

```typescript
interface ActivityStats {
  duration: number;                // 时长（毫秒），ongoing时实时计算
  totalRounds: number;           // 总轮数
  playerStats: PlayerStat[];     // 每个玩家的统计
  topPlayer: PlayerStat;         // 第一名
  maxRoundScore: {               // 单轮最大积分
    roundNumber: number;
    playerIndex: number;
    score: number;
  } | null;
}

interface PlayerStat {
  index: number;                 // 玩家序号
  nickname: string;
  color: string;
  totalScore: number;            // 总积分
  rank: number;                  // 排名（1开始）
}
```

### 3.3 存储方案

```javascript
// localStorage 键名
const STORAGE_KEY = 'scoretrack_activities';
```

## 4. 业务规则

### 4.1 积分规则
- 每轮积分必须平衡：所有玩家本轮积分之和 = 0
- 正数：玩家获得积分
- 负数：玩家扣除积分
- 零：无变化

### 4.2 排名规则
1. 总积分降序
2. 同分按玩家序号（先创建的在前）

### 4.3 自动结束规则
- 触发条件：`now - lastRoundAt > 1小时`
- 行为：自动将状态改为 `finished`，设置 `endedAt`
- 限制：已结束活动不可恢复

### 4.4 轮次限制
- 最大轮次：1000轮
- 达到限制：禁止新增轮次，提示"已达到最大轮次限制"

## 5. 界面需求

### 5.1 配色方案
- 支持双色模式：浅色 / 深色
- 默认：跟随系统（auto）
- 可手动切换

### 5.2 页面清单

| 页面 | 路由 | 说明 |
|------|------|------|
| 首页 | `/` | 活动列表 |
| 创建活动 | `/create` | 新建活动表单 |
| 活动概览 | `/activity/:id` | 进行中活动的概览 |
| 积分录入 | `/activity/:id/round` | 录入本轮分数 |
| 轮次详情 | `/activity/:id/rounds` | 查看/编辑所有轮次 |
| 活动结果 | `/activity/:id/results` | 已结束活动的概览 |

### 5.3 响应式
- 主要适配移动端（375px - 428px 宽度）
- 平板/桌面端居中显示，最大宽度 480px

## 7. 非功能需求

### 7.1 性能
- 首屏加载 < 2s
- 交互响应 < 100ms
- 支持1000轮数据流畅操作

### 7.2 可靠性
- 关键操作二次确认（结束活动、删除轮次）
- 数据自动保存到 localStorage
- 异常状态恢复（页面刷新后恢复当前活动）

### 7.3 可访问性
- 按钮最小点击区域 44x44px
- 颜色对比度符合 WCAG 2.1 AA 标准
- 支持系统字体大小设置
